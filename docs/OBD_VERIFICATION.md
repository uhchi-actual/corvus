## Read Scope

Corvus live capture uses only these python-OBD command names:

| Mode | python-OBD command names | Corvus use |
| --- | --- | --- |
| 01 | `RPM`, `SPEED`, `ENGINE_LOAD`, `COOLANT_TEMP`, `INTAKE_TEMP`, `MAF`, `THROTTLE_POS`, `SHORT_FUEL_TRIM_1`, `LONG_FUEL_TRIM_1`, `TIMING_ADVANCE`, `FUEL_PRESSURE`, `O2_B1S1` | Telemetry rows |
| 03 | `GET_DTC` | Stored trouble codes |
| 07 | `GET_CURRENT_DTC` | Pending/current-cycle trouble codes |
| 09 | `VIN` | Vehicle identity |

Mode 02 freeze-frame commands are generated by python-OBD as `DTC_` prefixed
Mode 01 names. Phase 1 does not persist freeze-frame rows yet; it is reserved
for a later expansion after the core CSV/emulator/live paths are stable.

## Excluded Write Scope

`CLEAR_DTC` exists in python-OBD as a Mode 04 command. Corvus does not call it,
does not expose it in adapter command lists, and treats clearing codes or ECU
writes as out of scope.
